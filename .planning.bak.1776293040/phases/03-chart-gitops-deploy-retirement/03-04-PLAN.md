```yaml
name: Portfolio Chart - Lint & Publish

on:
  push:
    branches: [main]
    paths:
      - 'HelmCharts/portfolio/**'
      - '.github/workflows/helmchart.yaml'
  pull_request:
    branches: [main]
    paths:
      - 'HelmCharts/portfolio/**'
  workflow_dispatch: {}

permissions:
  id-token: write
  contents: read

env:
  AWS_REGION: us-east-1
  ECR_REGISTRY: 372517046622.dkr.ecr.us-east-1.amazonaws.com
  CHART_DIR: HelmCharts/portfolio

concurrency:
  group: portfolio-chart-${{ github.ref }}
  cancel-in-progress: false

jobs:
  lint-publish:
    runs-on: ubuntu-22.04
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Helm
        uses: azure/setup-helm@v4
        with:
          version: v3.15.0

      - name: Helm lint
        run: helm lint ${{ env.CHART_DIR }}

      - name: Helm template (smoke render)
        run: helm template ${{ env.CHART_DIR }} > /tmp/rendered.yaml

      - name: Read chart version
        id: chart
        shell: bash
        run: |
          set -euo pipefail
          VERSION=$(awk '/^version:/ {print $2}' ${{ env.CHART_DIR }}/Chart.yaml | tr -d '"')
          APP=$(awk '/^appVersion:/ {print $2}' ${{ env.CHART_DIR }}/Chart.yaml | tr -d '"')
          echo "version=${VERSION}" >> "$GITHUB_OUTPUT"
          echo "appVersion=${APP}" >> "$GITHUB_OUTPUT"

      - name: Configure AWS Credentials (OIDC)
        if: github.event_name != 'pull_request'
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ vars.IAM_ROLE }}
          role-session-name: GitHubActionsPortfolioChart
          aws-region: ${{ env.AWS_REGION }}

      - name: Login to Amazon ECR
        if: github.event_name != 'pull_request'
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v2

      - name: Helm package
        if: github.event_name != 'pull_request'
        run: |
          set -euo pipefail
          helm package ${{ env.CHART_DIR }} --destination /tmp

      - name: Helm push to ECR OCI
        if: github.event_name != 'pull_request'
        shell: bash
        run: |
          set -euo pipefail
          PKG="/tmp/portfolio-${{ steps.chart.outputs.version }}.tgz"
          test -f "$PKG"
          helm push "$PKG" oci://${{ env.ECR_REGISTRY }}/helm-charts

      - name: Write chart info to job summary
        if: github.event_name != 'pull_request'
        shell: bash
        run: |
          {
            echo "### portfolio chart"
            echo ""
            echo "- version: \`${{ steps.chart.outputs.version }}\`"
            echo "- appVersion: \`${{ steps.chart.outputs.appVersion }}\`"
            echo "- registry: \`oci://${{ env.ECR_REGISTRY }}/helm-charts/portfolio\`"
          } >> "$GITHUB_STEP_SUMMARY"
```

Key changes vs old file:
- Lint + smoke-template BEFORE any AWS steps (fail fast without consuming AWS quota).
- Path filter narrowed from `HelmCharts/**` to `HelmCharts/portfolio/**` (D-06).
- PR builds run lint only (no push) — matches portfolio-images.yaml's pattern.
- Concurrency group `portfolio-chart-${{ github.ref }}` with `cancel-in-progress: false` (D-06) — critical because ECR treats the same version tag as immutable and parallel pushes would fail one of them.
- Chart version extracted and echoed to step summary so operators can copy/paste into a HelmRelease pin if needed.
- Explicit helm binary version pinned (v3.15.0) for reproducibility.
- Package goes to /tmp then is pushed by exact path — avoids glob ambiguity if older artifacts exist in the workdir.

Why NOT a sha-suffixed version: D-11 says `Chart.yaml version` is the version; ECR refuses to re-push the same version — so the human workflow is "bump Chart.yaml, commit, push". Plan 03-06 formalises this. Auto-bumping per-sha would clutter ECR and is explicitly not in D-06.

**In `variables.tf`** -- add a new `egress_rules` variable after the existing `rules` variable. Use the identical type structure:

```hcl
variable "egress_rules" {
  type = list(object({
    cidr       = string
    from_port  = number
    to_port    = number
    protocol   = optional(string, "tcp")
    ip_version = optional(string, "ipv4")
  }))
  default     = []
  description = "List of egress rules for the security group. Empty list means no explicit egress."
}
```

**In `security-group.tf`** -- make two changes:
1. DELETE the two hardcoded wide-open egress resources (`aws_vpc_security_group_egress_rule.all_ipv4` and `aws_vpc_security_group_egress_rule.all_ipv6`).
2. ADD a new dynamic egress resource using `for_each`, mirroring the existing ingress pattern exactly:

```hcl
# Dynamically create egress rules (mirrors ingress pattern)
resource "aws_vpc_security_group_egress_rule" "sg" {
  for_each          = { for idx, rule in var.egress_rules : idx => rule }
  security_group_id = aws_security_group.sg.id
  from_port         = each.value.from_port
  to_port           = each.value.to_port
  ip_protocol       = each.value.protocol

  cidr_ipv4 = each.value.ip_version == "ipv4" ? each.value.cidr : null
  cidr_ipv6 = each.value.ip_version == "ipv6" ? each.value.cidr : null
}
```

Keep the existing `aws_security_group.sg` resource and `aws_vpc_security_group_ingress_rule.sg` resource unchanged.

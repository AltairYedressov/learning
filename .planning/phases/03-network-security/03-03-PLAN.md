**Create `platform-tools/istio/istio-ingress/base/networkpolicy.yaml`:**
The ingress gateway receives external traffic and routes to application pods. It is NOT Istio-injected itself (it IS the Envoy proxy).

```yaml
# Default-deny all ingress/egress to istio ingress gateway,
# then allow only what it strictly needs.
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: istio-ingressgateway
  namespace: istio-ingress
spec:
  podSelector:
    matchLabels:
      istio: ingressgateway
  policyTypes:
    - Ingress
    - Egress
  ingress:
    # Allow external traffic via NLB (HTTP redirect + HTTPS)
    - ports:
        - protocol: TCP
          port: 8080
        - protocol: TCP
          port: 8443
  egress:
    # DNS resolution
    - ports:
        - protocol: UDP
          port: 53
        - protocol: TCP
          port: 53
    # Route traffic to application pods in portfolio namespace
    - to:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: portfolio
      ports:
        - protocol: TCP
          port: 8000
        - protocol: TCP
          port: 3000
    # istiod xDS configuration
    - to:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: istio-system
      ports:
        - protocol: TCP
          port: 15012
    # kube-apiserver (for service discovery)
    - ports:
        - protocol: TCP
          port: 443
        - protocol: TCP
          port: 6443
```

**Create `platform-tools/istio/istio-system/base/networkpolicy.yaml`:**
istiod serves xDS to all sidecars and manages certificates. It is the control plane -- NOT sidecar-injected.

```yaml
# Default-deny all ingress/egress to istiod,
# then allow only what it strictly needs.
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: istiod
  namespace: istio-system
spec:
  podSelector:
    matchLabels:
      app: istiod
  policyTypes:
    - Ingress
    - Egress
  ingress:
    # Allow xDS connections from all Istio-injected namespaces
    - ports:
        - protocol: TCP
          port: 15012
    # Allow webhook calls from kube-apiserver (admission webhooks)
    - ports:
        - protocol: TCP
          port: 15017
    # Allow Prometheus metrics scraping
    - from:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: monitoring
      ports:
        - protocol: TCP
          port: 15014
  egress:
    # DNS resolution
    - ports:
        - protocol: UDP
          port: 53
        - protocol: TCP
          port: 53
    # kube-apiserver (for config watches, webhook registration)
    - ports:
        - protocol: TCP
          port: 443
        - protocol: TCP
          port: 6443
```

Note: istiod ingress on port 15012 uses an empty `from:` (no selector) to allow all namespaces to connect. This is intentional -- any pod with an Istio sidecar needs xDS access regardless of namespace. Same for port 15017 (webhook) which is called by the API server from any source IP.

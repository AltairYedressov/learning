**Create `platform-tools/istio/istio-ingress/base/networkpolicy.yaml`:**
The ingress gateway receives external traffic and routes to application pods. It is NOT Istio-injected itself (it IS the Envoy proxy).

```yaml
# Default-deny all ingress/egress to istio ingress gateway,
# then allow only what it strictly needs.
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: istio-ingressgateway
  namespace: istio-ingress
spec:
  podSelector:
    matchLabels:
      istio: ingressgateway
  policyTypes:
    - Ingress
    - Egress
  ingress:
    # Allow external traffic via NLB (HTTP redirect + HTTPS)
    - ports:
        - protocol: TCP
          port: 8080
        - protocol: TCP
          port: 8443
  egress:
    # DNS resolution
    - ports:
        - protocol: UDP
          port: 53
        - protocol: TCP
          port: 53
    # Route traffic to application pods in portfolio namespace
    - to:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: portfolio
      ports:
        - protocol: TCP
          port: 8000
        - protocol: TCP
          port: 3000
    # istiod xDS configuration
    - to:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: istio-system
      ports:
        - protocol: TCP
          port: 15012
    # kube-apiserver (for service discovery)
    - ports:
        - protocol: TCP
          port: 443
        - protocol: TCP
          port: 6443
```

**Create `platform-tools/istio/istio-system/base/networkpolicy.yaml`:**
istiod serves xDS to all sidecars and manages certificates. It is the control plane -- NOT sidecar-injected.

```yaml
# Default-deny all ingress/egress to istiod,
# then allow only what it strictly needs.
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: istiod
  namespace: istio-system
spec:
  podSelector:
    matchLabels:
      app: istiod
  policyTypes:
    - Ingress
    - Egress
  ingress:
    # Allow xDS connections from all Istio-injected namespaces
    - ports:
        - protocol: TCP
          port: 15012
    # Allow webhook calls from kube-apiserver (admission webhooks)
    - ports:
        - protocol: TCP
          port: 15017
    # Allow Prometheus metrics scraping
    - from:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: monitoring
      ports:
        - protocol: TCP
          port: 15014
  egress:
    # DNS resolution
    - ports:
        - protocol: UDP
          port: 53
        - protocol: TCP
          port: 53
    # kube-apiserver (for config watches, webhook registration)
    - ports:
        - protocol: TCP
          port: 443
        - protocol: TCP
          port: 6443
```

Note: istiod ingress on port 15012 uses an empty `from:` (no selector) to allow all namespaces to connect. This is intentional -- any pod with an Istio sidecar needs xDS access regardless of namespace. Same for port 15017 (webhook) which is called by the API server from any source IP.

**Create `clusters/dev-projectx/flux-system/networkpolicy.yaml`:**
Flux controllers need: GitHub API (HTTPS 443 to internet), kube-apiserver (443/6443), DNS (53). Per Pitfall 3, blocking Flux egress breaks all GitOps reconciliation.

```yaml
# Default-deny all ingress/egress to Flux controllers,
# then allow only what they strictly need.
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: flux-controllers
  namespace: flux-system
spec:
  podSelector: {}
  policyTypes:
    - Ingress
    - Egress
  ingress:
    # Allow webhook receivers (notification-controller)
    - ports:
        - protocol: TCP
          port: 9292
    # Allow Prometheus metrics scraping
    - from:
        - namespaceSelector:
            matchLabels:
              kubernetes.io/metadata.name: monitoring
      ports:
        - protocol: TCP
          port: 8080
  egress:
    # DNS resolution
    - ports:
        - protocol: UDP
          port: 53
        - protocol: TCP
          port: 53
    # GitHub API, Helm repos, OCI registries (HTTPS to internet)
    - ports:
        - protocol: TCP
          port: 443
    # kube-apiserver
    - ports:
        - protocol: TCP
          port: 6443
    # SSH for Git over SSH (if used)
    - ports:
        - protocol: TCP
          port: 22
```

Note: flux-system uses `podSelector: {}` to cover ALL pods in the namespace (source-controller, kustomize-controller, helm-controller, notification-controller). They all need the same egress (GitHub + kube-apiserver). Port 443 egress is open to all destinations (no `to:` selector) because Flux fetches from GitHub, Helm repos, and OCI registries at various IPs.

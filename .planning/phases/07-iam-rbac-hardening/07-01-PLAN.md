Three changes, all per user decisions D-01 through D-05:

**1. Create `terraform-infra/iam-role-module/Policies/eks_worker_node_policy.json`:**

New file with this exact content (per D-02 -- minimal custom policy for node metadata lookups):
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowNodeMetadataLookup",
      "Effect": "Allow",
      "Action": [
        "ec2:DescribeInstances",
        "ec2:DescribeTags"
      ],
      "Resource": "*"
    }
  ]
}
```

Note: `Resource: "*"` is required because ec2:Describe* actions do not support resource-level restrictions in IAM. `ec2:DescribeInstances` and `ec2:DescribeTags` are the minimum for EKS node self-registration. The CNI plugin's ec2:DescribeNetworkInterfaces is already covered by `AmazonEKS_CNI_Policy`.

**2. Update `terraform-infra/root/dev/iam-roles/variables.tf`:**
